# CLAUDE.md - 项目规则

## 功能文档同步

**每次新增或修改网站功能后，必须同步更新 `README.md` 中的"功能"列表。**

具体做法：
- 新增功能 → 在 README.md "功能"部分添加一条描述
- 修改现有功能 → 更新对应的描述文字
- 新增配置项 → 更新"配置"表格
- 修改项目结构 → 更新"项目结构"目录树

**Why**: README.md 是这个项目唯一的功能文档，保持它与代码同步，避免后续维护时功能遗漏或描述过时。

## 架构文档同步

**每次修改代码结构（新增/删除/重命名文件、更换技术方案、改动数据流）后，必须同步更新 `ARCHITECTURE.md`。**

具体做法：
- 文件变更 → 更新"文件职责"目录树
- 技术栈变化 → 更新"技术栈"表格
- 数据流变化 → 更新"数据流"章节
- 部署方式变化 → 更新"部署流水线"章节

**Why**: ARCHITECTURE.md 是新开发者（或未来的自己）快速理解项目的入口。结构变化不更新 = 文档腐烂。

## 复现提示词同步

**每次新增功能、修改样式、调整架构后，必须同步更新 `PROMPT.md`。**

具体做法：
- 新增功能 → 在"功能清单"中添加一项
- 修改配色/样式 → 更新"配色方案"
- 调整技术方案 → 更新"技术栈"或"CloudBase 配置"
- 修改 config.js 结构 → 更新"config.js 结构"

**Why**: PROMPT.md 是其他人复现这个网站的提示词模板。保持与网站实际状态一致，别人才能一键复现。
